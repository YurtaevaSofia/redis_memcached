## Задание 1. Кеширование

**Примеры проблем, которые решает кеширование:**

1. **Высокая нагрузка на базу данных.** При большом количестве одинаковых запросов (например, главная страница сайта читается тысячи раз в минуту) БД испытывает избыточную нагрузку. Кеш позволяет отдавать уже готовый результат без обращения к БД.

2. **Медленное время отклика.** Сложные SQL-запросы с JOIN-ами и агрегациями могут выполняться секунды. Закешировав результат, последующие запросы получают ответ за миллисекунды.

3. **Повторные вычисления.** Если приложение регулярно пересчитывает одни и те же данные (статистика, рейтинги, рекомендации), кеш позволяет хранить уже вычисленный результат и не тратить ресурсы процессора на повторную работу.

4. **Перегрузка при всплесках трафика.** Во время пиковой нагрузки (распродажи, новостные события) поток запросов резко возрастает. Кеш принимает на себя большую часть нагрузки и защищает backend-сервисы от перегрева.

5. **Дорогие обращения к внешним API.** Сторонние сервисы часто имеют лимиты запросов и берут деньги за превышение. Кеширование ответов внешних API сокращает число реальных вызовов и экономит бюджет.

6. **Медленные сетевые операции.** Данные, которые физически хранятся далеко (другой датацентр, удалённая файловая система), долго передаются по сети. Кеш на стороне приложения устраняет эту задержку.

7. **Сессии пользователей.** Хранение данных сессий в памяти (а не в БД) ускоряет аутентификацию и снижает нагрузку на постоянное хранилище.
